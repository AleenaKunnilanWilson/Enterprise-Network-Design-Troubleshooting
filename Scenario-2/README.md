# 🌐 Scenario 2 – Enterprise Network Security with RIPv2 and Access Control Lists

## 📖 Overview

This scenario focuses on designing and configuring a secure enterprise network using Cisco Packet Tracer. Building upon the previous scenario, it expands the network by introducing an additional router and switch while implementing Access Control Lists (ACLs) to enforce network security policies. The project reinforces enterprise networking concepts including VLSM subnetting, VLANs, inter-VLAN routing, RIPv2 dynamic routing, static routing, and traffic filtering using extended ACLs.

---

## 📋 Scenario Requirements

In this scenario, the objective was to design and configure an enterprise network consisting of four routers and two switches using Cisco Packet Tracer.

The main requirements included:

- Design the network based on the provided topology.
- Perform VLSM subnetting for the corporate network.
- Configure VLANs and inter-VLAN routing.
- Configure router and switch interfaces.
- Configure switch management, port security, and Telnet access.
- Implement static routing between the enterprise gateway and ISP.
- Configure RIPv2 between the internal routers.
- Configure extended Access Control Lists (ACLs) to enforce network security policies.
- Verify routing functionality and end-to-end network connectivity.

---

## ⚙️ Implementation

- Designed the enterprise network topology.
- Calculated and assigned IP addresses using VLSM.
- Configured VLANs, router subinterfaces, and trunk links.
- Configured router and switch interfaces.
- Applied switch management settings and port security.
- Configured static routes and implemented RIPv2 dynamic routing.
- Designed and implemented extended ACLs to control HTTP and IP traffic between internal VLANs and external networks.
- Verified ACL operation, routing tables, and end-to-end connectivity.
