# 🌐 Scenario 1 – Basic Enterprise Network with RIPv2

## 📖 Overview

This scenario focuses on designing and configuring a basic enterprise network using Cisco Packet Tracer. It introduces dynamic routing with RIPv2 while reinforcing networking concepts such as subnetting, VLANs, inter-VLAN routing, and static routing. :contentReference[oaicite:0]{index=0}

---

## 📋 Scenario Requirements

In this scenario, the objective was to design and configure an enterprise network consisting of three routers and one switch using Cisco Packet Tracer.

The main requirements included:

- Design the network based on the provided topology.
- Perform VLSM subnetting for the corporate network.
- Configure VLANs and inter-VLAN routing.
- Configure router and switch interfaces.
- Configure switch management and port security.
- Implement static routing between the enterprise gateway and ISP.
- Configure RIPv2 on the internal routers.
- Verify end-to-end connectivity across the network.

---

## ⚙️ Implementation

- Designed the enterprise network topology
- Calculated and assigned IP addresses using VLSM
- Configured VLANs and router subinterfaces
- Configured router and switch interfaces
- Applied port security and switch management settings
- Configured static routes and implemented RIPv2
